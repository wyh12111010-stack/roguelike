## 〇、同类型游戏参考与可行建议

### 参考案例

| 游戏 | 敌人差异化方式 | 可借鉴点 |
|------|----------------|----------|
| **Hades** | 每区域（Tartarus/Asphodel/Elysium）专属敌人池；敌人行为与环境匹配（如 Asphodel 敌人会跨岩浆） | 大关 = 专属敌人池，身份/行为与场景一致 |
| **Dead Cells** | 每 biome 敌人池不同；普通/精英变体；难度影响出现频率与分布 | 同类型可做精英变体（血×1.5、攻×1.2），不新增美术 |
| **Vampire Survivors** | 随时间/波次逐步解锁新敌人；从落单僵尸→兽人/蝙蝠群→死神 | 类型逐步解锁，前期少、后期多 |
| **Tiny Rogues** | 职业/装备多样，敌人配合 arena 战斗；随机生成制造组合变化 | 有限基础类型 + 随机组合 = 丰富体验 |

### 通用原则（Roguelike 设计）

- **张力与推理**：敌人制造可理解的挑战，而非纯数值碾压
- **类型有战略意义**：每种敌人与玩家资源/法宝有不同互动，避免「纯废」或「纯强」
- **有限类型 + 组合**：少量基础类型，通过组合与随机生成产生变化

### 可行建议（结合本项目）

| 建议 | 实现难度 | 说明 |
|------|----------|------|
| **1. 大关专属敌人池** | 中 | 试炼只出 melee/ranged；秘境加入 charge/aoe；凶地加入 homing/summon。改 `levels.json` 配置即可，无需新美术 |
| **2. 身份换皮（三大关）** | 高 | 同类型在不同大关用不同身份：试炼=修士、秘境=妖兽、凶地=秽灵。需 6 类型×3=18 种美术，可分批做 |
| **3. 精英变体（无新美术）** | 低 | 精英节点用同精灵图，仅调血攻。Dead Cells 做法，零美术成本 |
| **4. 类型逐步解锁** | 低 | 与 1 一致，通过配置限制每大关可用类型 |
| **5. 混合方案（推荐）** | 中 | 1+3：大关专属池 + 精英用同图。先落地玩法差异，美术换皮后续迭代 |

## 〇·二、敌人行为与逻辑（讨论）

> 同类型游戏不是「近战=直冲」，而是**类型（身体）与行为（逻辑）分离**，组合出丰富变化。

### 参考：类型 × 行为 分离（Game Developer）

**问题**：若「近战」= 一种行为，则所有近战都直愣愣冲过来；若要近战有巡逻、冲锋、打带跑，就得写 N 种近战类 → 组合爆炸。

**做法**：把**敌人类型**（血、攻、外观、攻击方式）和**行为逻辑**（移动、决策）拆开。同一类型可配不同行为，同一行为可配不同类型。

```
类型（身体）    行为（AI）
MeleeEnemy  +  ChaseBehavior      → 直冲近战
MeleeEnemy  +  PatrolThenChase    → 巡逻发现后追击
MeleeEnemy  +  HitAndRunBehavior  → 打一下就跑
RangedEnemy +  KiteBehavior       → 保持距离射击（当前已有）
RangedEnemy +  StandAndShoot      → 站桩射击
```

### 常见行为模式（同类型游戏）

| 行为 | 移动逻辑 | 攻击逻辑 | 适用类型 |
|------|----------|----------|----------|
| **Chase** | 持续向玩家移动 | 接触即攻击 | 近战（当前 melee） |
| **Patrol → Chase** | 沿路径巡逻，发现玩家后追击 | 接触即攻击 | 近战、可做「站岗型」 |
| **Hit and Run** | 接近→攻击→后撤→再接近 | 单次攻击后冷却+后退 | 近战，增加节奏感 |
| **Lunge** | 平时慢移，进入范围后短距冲刺 | 冲刺末端造成伤害 | 近战，类似 charge 但更短 |
| **Kite** | 保持固定距离，远了靠近、近了远离 | 远程射击 | 远程（当前 ranged） |
| **Stand and Shoot** | 不移动或微移，站桩输出 | 远程射击 | 远程，炮台型 |
| **Charge** | 蓄力后直线冲刺 | 冲刺路径造成伤害 | 突进（当前 charge） |

### 状态机示例（Patrol → Alert → Attack）

参考 Ben Stewart、Hades 等：

1. **Patrol**：沿路径/随机游走，未发现玩家
2. **Alert**：发现玩家（视野/距离），开始追击
3. **Attack**：进入攻击范围，执行攻击
4. **Lost**：失去目标（玩家离开视野），回到 Patrol

当前项目：多数敌人无 Patrol，直接 Chase。可先为部分敌人加「发现距离」或「巡逻路径」。

### 近战变体举例（同 melee 类型，不同行为）

| 变体 | 行为 | 玩家感受 |
|------|------|----------|
| 莽夫 | Chase，一直冲 | 压力大，需走位 |
| 巡逻兵 | Patrol → Chase | 有节奏，可预判 |
| 刺客 | Hit and Run | 打一下就跑，需追击 |
| 重击手 | 慢移 + 短距 Lunge | 有前摇，可闪避 |

### 远程变体举例

| 变体 | 行为 | 玩家感受 |
|------|------|----------|
| 风筝 | Kite（当前 ranged） | 保持距离，需追或绕 |
| 炮台 | Stand and Shoot | 站桩输出，需优先清理 |
| 游击 | 移动中射击，不严格保持距离 | 更难预判 |

### 落地建议（结合当前 enemy.py）

| 阶段 | 做法 | 工作量 |
|------|------|--------|
| **1. 行为参数化** | 在 `enemy_type` 或配置中加 `behavior` 字段，如 `melee_chase`、`melee_hitrun` | 低 |
| **2. 行为模块化** | 抽离 `_move_xxx`、`_attack_xxx` 为可组合函数/类，Enemy 按 behavior 调用 | 中 |
| **3. 新增行为** | 实现 HitAndRun、PatrolThenChase 等，先做 1–2 种验证 | 中 |
| **4. 配置驱动** | `levels.json` 中每敌人可指定 `behavior`，与 `type` 解耦 | 低 |

**建议**：先做 1+3，为 melee 增加 `melee_hitrun`（打带跑）变体，验证「同类型不同行为」的可行性，再扩展行为库与配置。

**✅ 已实现**：`behavior` 参数化、`MeleeHitRunEnemy`、`levels.json` 配置驱动。见 `docs/PROJECT_ROADMAP.md`。

## 三、美术：动画状态建议

### 3.1 状态数量（按制作成本）

| 方案 | 状态 | 帧数 | 适用 |
|------|------|------|------|
| **最小** | 待机 | 1 帧 | 快速出图 |
| **推荐** | 待机 + 死亡 | 1 帧 + 4 帧 | 与主角死亡风格一致 |
| **完整** | 待机 + 攻击 + 死亡 | 2 + 1 + 4 | 表现力最佳 |

### 3.2 推荐方案：待机 + 死亡

- **待机**：1 帧站立，或 2 帧轻微晃动（可选）
- **死亡**：4 帧倒地序列，与主角死亡模板一致（站立→后仰→下落→倒地）

理由：敌人数量多（6 种普通 + 4 种 Boss），每类做 2 种状态（待机、死亡）即可保证辨识度与节奏感；攻击可用粒子/特效补充。

## 七、能力与逻辑规范（Balanced）

> 本节用于统一敌人能力逻辑模板，支持后续扩展而不失控。

### 7.1 敌人设计目标

- 让玩家“看懂威胁 -> 做出反制 -> 获得收益”。
- 普通敌人强调单一职责，Boss 才做多线程压迫。
- 数值增强应晚于行为增强。

### 7.2 统一能力模板（类型与 Boss 都适用）

| 维度 | 内容 |
|------|------|
| 定位 | 该敌人主要施压方式（追击/封位/耗损/召唤） |
| 前摇 | 攻击启动提示（时间与视觉） |
| 执行 | 生效范围、持续时间、伤害 |
| 后摇 | 玩家反打窗口 |
| 冷却 | 频率控制 |
| 克制关系 | 被什么玩法克制 |
| 风险点 | 过强或过弱时的问题 |

### 7.3 普通敌人职责分工

| 类型 | 主要职责 | 玩家优先级 |
|------|----------|------------|
| melee | 贴身追击 | 中 |
| ranged | 远程压血 | 高 |
| charge | 位移惩罚 | 高 |
| aoe | 封锁地面 | 中高 |
| homing | 持续逼位 | 高 |
| summon | 延长战线 | 最高 |

### 7.4 行为窗口规范

- 近战冲脸类：攻击前摇 >= 0.15s。
- 冲刺/突进类：前摇 >= 0.35s，结束后有后摇。
- AOE 类：预警圈先于伤害判定 >= 0.35s。
- 召唤类：召唤动作期间本体应降低压迫（避免无解叠压）。

### 7.5 压迫上限规则

为避免“不公平死亡”，同屏压力遵循：

1. 同时在线召唤单位设上限。  
2. 追踪弹与大范围 AOE 不连续无限叠加。  
3. 高速突进技能不能无前摇。  
4. 召唤与再生必须有限次（Boss 已拍板 <=2 次）。  

### 7.6 反模式（必须避免）

- 纯追踪+高伤+高频，几乎不可躲。
- 无后摇高爆发，玩家无反打窗口。
- 小怪拥有 Boss 级多段机制，破坏层级。
- 敌人视觉提示与实际判定不一致。

### 7.7 验收清单（敌人逻辑）

- 玩家能在 1~2 次遭遇内识别该敌人的主要威胁。  
- 该敌人至少有一个稳定反制方案。  
- 与其他敌人组合时不会形成不可解局面。  
- 失败可归因：站位、优先级、资源管理，而非读不懂。
