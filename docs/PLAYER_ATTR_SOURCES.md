# 属性来源与优先级规范

> 目标：把属性系统从“概念描述”升级为“可执行规则”。  
> 本文定义属性来源、触发顺序、叠加上限、冲突处理和反馈标准。

---

## 一、属性分层模型

### 1.1 三层结构

1. **基础层（Base）**：角色白板属性（血、蓝、速、基础伤害）。  
2. **构筑层（Build）**：灵根、法宝、饰品、伙伴印记带来的持续修正。  
3. **战斗层（Combat）**：短时状态（减速、虚弱、DOT、增伤）与元素反应。  

计算原则：先 Base -> 再 Build -> 最后 Combat。

### 1.2 当前基线（代码口径）

| 属性 | 当前值 |
|------|--------|
| max_health | 100 (+ `base_health_bonus` 预留) |
| max_mana | 100 (+ `base_mana_bonus` 预留) |
| speed | 200 |
| invincible_time_on_hit | 0.8s |
| mana_regen | 3/s |
| enemy_speed_cap | 175 |

## 二、伤害与反应优先级

### 2.1 伤害计算顺序（玩家 -> 敌人）

```text
baseDamage
-> weaponMultiplier（法宝档位）
-> resonanceOrReactionMultiplier（相合/相异）
-> additiveBonus（饰品/伙伴等平加）
-> defensiveReduction（目标减伤/抗性，若启用）
-> finalDamage
```

### 2.2 相合/反应优先级

| 条件 | 优先级 | 结果 |
|------|--------|------|
| 灵根=法宝属性 | 最高 | 触发相合倍率（1.20） |
| 灵根与法宝可反应 | 次级 | 触发反应倍率（1.10） |
| 其他 | 最低 | 1.00 |

同一次命中只取一种倍率，不允许相合与反应叠乘。

### 2.3 二层反应触发（五行效果）

- 二层反应在“一层倍率”之后判定。
- 每次攻击最多触发一个二层效果。
- 对同一目标设置触发间隔（建议 0.2~0.4s）防止多段攻击爆刷。

## 三、状态叠加与上限规则

### 3.1 速度类状态

| 状态 | 叠加方式 | 上限建议 |
|------|----------|----------|
| 减速 | 取最大值 | 最大 40% |
| 加速 | 取最大值 | 最大 35% |
| 减速+加速同存 | 先减后加（乘算） | 结果最低不低于基础速 50% |

### 3.2 伤害类状态

| 状态 | 叠加方式 | 上限建议 |
|------|----------|----------|
| 增伤 | 同类取最大，不同类乘算 | 总增伤不超过 +120% |
| 减伤 | 同类取最大，不同类乘算 | 总减伤不超过 60% |
| DOT | 同名刷新时长，不叠层伤害 | 多异名可并存 |

### 3.3 控制类状态

- 短控制（击退、硬直）优先级高于移动，但不应锁死玩家。
- 对同一目标设置控制衰减（短时间重复命中逐步缩短控制）。

## 四、属性冲突处理

### 4.1 冲突优先级

1. 死亡结算  
2. 无敌判定  
3. 伤害结算  
4. 触发反应  
5. 施加状态  

### 4.2 必须避免的冲突

- 同帧既判定无敌又吃伤害。
- DOT 在目标死亡后继续结算并触发额外收益。
- 反应效果在 0 伤害命中时仍生效（应禁止或特殊标记）。

---

## 五、反馈规范（玩家能读懂）

### 5.1 视觉反馈

| 事件 | 建议反馈 |
|------|----------|
| 相合触发 | 金色/对应属性强化数字 |
| 反应触发 | 元素关键字飘字（如“灭火”） |
| DOT 生效 | 小号持续跳字 |
| 虚弱/减速 | 目标脚底环或图标 |

### 5.2 数值反馈

- 暴击/高额伤害与普通伤害区分颜色与字号。
- 反应伤害与本体伤害分开显示或带标签。

---

## 六、来源清单（按实现优先级）

### 6.1 已有来源（当前）

- 灵根、法宝（一层倍率）
- 敌人弱控与 DOT（部分）
- 局内货币增减（灵石）

### 6.2 预留来源（后续）

- 伙伴印记
- 饰品被动
- 局外成长（`base_health_bonus` / `base_mana_bonus`）

---

## 七、验收清单（属性系统）

1. 同一次命中不会同时吃到相合与反应倍率。  
2. DOT 不会无限叠层。  
3. 减速与加速并存时结果稳定且可预测。  
4. 所有反应都有可读反馈。  
5. 回归测试下，伤害波动在可解释范围内。
