#### 1.1 法宝扩充（2 天）

**现状**：7 个法宝（剑/符/杖/刃/印/针/鼎）  
**目标**：扩充到 15-20 个，覆盖更多攻击形态和节奏

**新增方向**：
- **极速流**：攻速快、伤害低、连击感强
- **重压流**：攻速慢、伤害高、一击制敌
- **范围流**：AOE 大、适合清小怪
- **单体流**：单发高伤、适合打 Boss
- **特殊机制**：弹反、吸血、蓄力、连锁等

#### 1.3 Boss 机制强化（2 天）

**现状**：Boss 主要靠数值区分，缺少专属招式  
**目标**：4 个 Boss 各有 3-4 个专属技能，战斗体验差异化

**设计方向**：
- **妖王**：近战冲锋型，猛扑/火爪/震地/二连扑
- **剑魔**：灵活剑术型，剑气/瞬步/剑阵/剑雨
- **丹魔**：范围控制型，丹火/毒雾/爆炸/召唤丹傀
- **秽源**：终极 Boss，多阶段，融合前三者特点

**实现要点**：
- 每个技能有明显前摇（>0.3s）
- 技能之间有节奏变化（快慢结合）
- 阶段转换有明显提示（血量阈值）
- Boss 房有特殊音效/视觉提示

**验收标准**：
- [ ] 4 个 Boss 各有 3+ 个专属技能
- [ ] 每个技能有清晰的前摇和范围提示
- [ ] 通过 `fairness_gate.py` 检查
- [ ] 更新 `docs/BOSS_DESIGN.md` 实现细节

#### 1.4 成就系统补充（1 天）

**现状**：框架完整，但条目稀少  
**目标**：至少 40 个成就，覆盖各种玩法

**成就类型**：
- **进度类**：首次通关、击败各 Boss、通关 N 次
- **挑战类**：无伤通关、限时通关、特定条件通关
- **收集类**：解锁所有灵根/法宝、收集所有饰品
- **战斗类**：击杀 N 个敌人、触发 N 次反应、单局伤害达到 X
- **探索类**：访问所有节点类型、完成所有事件
- **流派类**：用特定灵根/法宝通关、纯近战/远程通关
- **经济类**：单局获得 N 灵石、购买 N 次商品

**验收标准**：
- [ ] 成就总数达到 40+ 个
- [ ] 覆盖进度/挑战/收集/战斗/探索/流派/经济 7 种类型
- [ ] 每个成就有清晰的解锁条件和奖励
- [ ] 更新 `achievement.py` 和 `docs/ACHIEVEMENT_TODO.md`

### 阶段 2：数值平衡（4-5 天）

**目标**：通过系统测试验证数值，确保难度曲线合理

#### 2.1 自动化测试（1 天）

**任务**：
- 扩展 `balance_test.py`，支持批量测试
- 编写测试脚本，模拟 50+ 局战斗
- 记录每局数据：时长、击杀数、死亡原因、资源消耗

**产出**：
- `tools/batch_test.py` - 批量测试脚本
- `tools/analyze_results.py` - 数据分析脚本
- 测试数据 CSV 文件

#### 2.2 难度曲线调优（2 天）

**测试目标**：
- 章节 1（试炼）：新手通过率 70%+，平均 8-12 分钟
- 章节 2（秘境）：中期通过率 50-60%，平均 10-15 分钟
- 章节 3（凶地）：终局通过率 30-40%，平均 12-18 分钟
- 全流程通关率：25-40%，平均 28-40 分钟

**调整方向**：
- 敌人血量/伤害
- 敌人密度和刷新节奏
- Boss 技能伤害和频率
- 玩家基础属性（如需要）

**验收标准**：
- [ ] 完成至少 50 局完整测试
- [ ] 通关率在 25-40% 区间
- [ ] 平均单局时长 28-40 分钟
- [ ] 各章节通过率符合目标
- [ ] 通过 `phase2_gate.py` 检查

#### 2.3 经济平衡（1 天）

**测试目标**：
- 单局平均灵石够买 2-3 件核心物品
- 3-5 局可解锁 1 个新灵根/法宝
- 商店价格与获取难度匹配
- 10 局内可解锁核心内容 50%

**调整方向**：
- 灵石产出（击杀/通关/精英）
- 商店价格（法宝/饰品/道韵碎片）
- 道韵产出（精英/Boss/凯旋）
- 局外成长价格

**验收标准**：
- [ ] 经济系统无明显短板或溢出
- [ ] 玩家有足够的购买选择
- [ ] 局外成长节奏合理
- [ ] 通过 `phase2_gate.py` 经济检查

#### 2.4 流派平衡（1 天）

**测试目标**：
- 各灵根通关率差异 <15%
- 各法宝通关率差异 <20%
- 快速/重击/范围流派都有生存空间

**调整方向**：
- 弱势灵根/法宝的数值
- 饰品对流派的支撑
- 敌人设计对流派的友好度

**验收标准**：
- [ ] 没有明显废弃的灵根/法宝
- [ ] 各流派都有对应的饰品支撑
- [ ] 玩家有多种通关路径
