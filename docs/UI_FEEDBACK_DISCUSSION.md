# UI 反馈讨论要点

> 待与策划/美术讨论后实现。以下为可考虑的 UI 反馈方向。

---

## 一、战斗内反馈

| 项目 | 现状 | 可改进方向 |
|------|------|------------|
| **血条** | 左上角文字「生命: x/y」 | 血条条、受伤闪红、低血警示 |
| **灵力条** | 左上角文字「灵力: x/y」 | 灵力条、施法消耗动画 |
| **技能 CD** | 无显式 CD 条 | 法宝 E 技能 CD 条、冷却中灰显 |
| **伙伴充能条** | 已有简单条 | 充能满时高亮/音效、技能就绪提示 |
| **受击反馈** | 无 | 屏幕震动、受击闪白、伤害数字飘字 |
| **击杀反馈** | 无 | 击杀飘字、连杀提示 |

---

## 二、结算与过渡

| 项目 | 现状 | 可改进方向 |
|------|------|------------|
| **过关结算** | 直接进入选路 | 过关弹窗（灵石/道韵获得）、过渡动画 |
| **选路界面** | 传送门样式 | 传送门动画、选中高亮、奖励图标 |
| **凯旋/身陨** | 简单文字 + 回村按钮 | 结算面板、统计汇总、回放/重试入口 |

---

## 三、村子与成长

| 项目 | 现状 | 可改进方向 |
|------|------|------------|
| **成就解锁** | 栖霞处查看 | 解锁时弹窗/Toast、成就音效 |
| **道韵/灵石** | 左上角数字 | 获得时 +N 飘字、消费时 -N 动画 |
| **伙伴治疗** | 点击即完成 | 治疗动画、羁绊升级提示 |

---

## 四、优先级建议

1. **高**：血条/灵力条可视化、受击反馈（提升战斗手感）
2. **中**：过关结算弹窗、成就解锁提示
3. **低**：过渡动画、飘字细节

---

## 五、技术实现参考

- 血条/灵力条：`game.py` 中 `_draw_ui` 已有文字，可改为 `pygame.draw.rect` 绘制条
- 飘字：可复用 `particle_mgr` 或新建 `FloatingText` 类
- 受击闪白：全屏半透明白色 overlay，短时淡出
