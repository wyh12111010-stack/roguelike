# Boss 战斗设计（成熟版）

> 目标：将 Boss 从“高血量小怪”升级为“可读、可学、可复盘、符合世界观”的高质量关底体验。  
> 适用范围：`segment_boss_1`、`segment_boss_2`、`segment_boss_3`、`final_boss`。

---

## 一、设计总目标（North Star）

### 1.1 玩家体验目标

- 每个 Boss 都有清晰“身份打法”：玩家 20 秒内能感知这个 Boss 在玩什么。
- 失败应可复盘：玩家能说出“我死于哪种前摇/区域/节奏”，而不是“莫名其妙暴毙”。
- 战斗有层次：开场学习 -> 中段压迫 -> 末段反杀窗口。
- 结算有成就感：击败 Boss 像“读懂机制并执行”，不只是“数值碾压”。

### 1.2 成熟 Boss 的四个硬指标

| 指标 | 目标 |
|------|------|
| **可读性** | 关键技能均有视觉前摇（>=0.3s），危险区域有明显提示 |
| **公平性** | 高伤技能至少留一个可靠解法（走位/绕后/拉距） |
| **差异性** | 每个 Boss 有 1 个“主机制” + 1 个“阶段变化” |
| **节奏性** | 每 8~12s 出现一次“高压窗口”，其间有 2~4s 回气窗口 |

## 二、同类游戏对标（提炼为本项目可执行规则）

> 不是照搬技能，而是提炼可落地的“战斗语法”。

| 参考游戏 | 可借鉴点 | 落地到本项目 |
|----------|----------|--------------|
| **Hades** | 明确前摇 + 招式循环 + 阶段强化 | Boss 技能固定冷却池，避免纯随机乱放 |
| **Dead Cells** | 阶段切换明显，场地压力提升 | 50%/40% 阈值切阶段，加入额外压迫技能 |
| **Skul** | 主体+随从联动，形成多线程压力 | 剑魔/秽源采用“主体技能 + 限次召唤/再生” |
| **Tiny Rogues** | 弹幕图形可读、冲刺预判强 | 火爪/秽弹幕采用固定扇角和弹速，不做不可躲散射 |

### 2.1 反模式（必须避免）

- 无前摇瞬发高伤。
- 召唤无限续杯导致战斗拖沓。
- 纯堆血量/伤害，不改变战斗决策。
- 屏幕信息过载（弹幕、AOE、冲刺同时爆发且无主次）。

## 三、世界观一致性（基于现有设定）

来源：`docs/WORLDVIEW_DESIGN.md`、`setting.py`。

- **妖王（虎形）**：被秽源侵染的妖兽王者，打法应体现“猛兽压迫”和“爆发扑杀”。
- **剑魔**：被侵染剑修，打法核心是“剑气控场 + 随从牵制”。
- **丹魔**：污染丹修，打法核心是“毒域经营 + 持续耗损”。
- **秽源**：污染源本体，打法应体现“秽能复生、压制节奏、后半段升级”。

设计原则：**机制先于特效**，特效服务于身份，不反过来。

## 四、通用 Boss 战斗语法（全 Boss 共享）

### 4.1 技能分类模板

- **A 类：位移压迫技**（冲刺/扑杀）  
  用于惩罚站桩和直线逃跑。
- **B 类：空间占领技**（AOE/毒环/弹幕扇形）  
  用于切割走位空间。
- **C 类：节奏变奏技**（召唤/再生/强化）  
  用于阶段转换与压力抬升。

每个 Boss 至少有 A+B，后两章 Boss 增加 C。

### 4.2 前摇与提示标准

| 技能类型 | 最低前摇 | 提示方式 |
|----------|----------|----------|
| 冲刺/扑杀 | 0.35s | 原地蓄力动作 + 朝向锁定 |
| 扇形弹幕 | 0.30s | 武器抬手 + 扇形方向显著 |
| 地面 AOE | 0.45s | 地面预警圈/渐亮 |
| 召唤/再生 | 0.80s | 吟唱状态，主体短暂低机动 |

### 4.3 压力与喘息节奏

- 连续高压时间不超过 6 秒。
- 高压后需给 1.5~3.0 秒喘息窗口（位移/输出机会）。
- 阶段切换瞬间不叠加多技能（避免“切阶段即秒杀”）。

### 4.4 反卡死规则

- 召唤或再生必须有上限（已拍板：2 次）。
- 冲刺技能结束后有短后摇，给玩家反击机会。
- 可避免不可解局面：同屏不可同时存在“高速追踪 + 大范围持续 AOE + 强制冲刺”三重叠加。

## 五、Boss 详细方案（正式版）

## 5.1 妖王（`segment_boss_1`）

**定位**：近身压迫型教学 Boss。  
**主机制**：扑杀链（单段扑 -> 二连扑）。  
**阶段阈值**：50%。

| 招式 | 作用 | 前摇 | 冷却 | 伤害 | 细节 |
|------|------|------|------|------|------|
| 猛扑 | 突进惩罚 | 0.4s | 2.2s | 18 | 冲刺 0.25s，速度 380 |
| 火爪 | 中距封位 | 0.3s | 2.5s | 8 x 3 | 60 度扇形，弹速 200 |
| 震地 | 近身驱离 | 0.5s | 3.0s | 12 | 半径 55 AOE |
| 二连扑 | 末段压迫 | 0.4s x 2 | 4.0s | 18 x 2 | 两扑间隔 0.5s（<50% 解锁） |

**行为意图**：
- P1：让玩家学习“预判冲刺 + 侧移规避扇形”。
- P2：引入二连扑，迫使玩家保留位移与节奏管理。

**当前状态**：已实现（`BossYaowang`）。

---

## 5.2 剑魔（`segment_boss_2`）

**定位**：多线程压力（主体 + 随从）。  
**主机制**：剑气直线压制与限次再召唤。  
**阶段阈值**：场上随从全灭后进入强化循环。

| 单位 | 招式 | 前摇 | 冷却 | 伤害 | 细节 |
|------|------|------|------|------|------|
| 剑魔主体 | 斩击 | 0.15s | 0.9s | 22 | 近战高压，贴身危险 |
| 剑魔主体 | 剑气 | 0.3s | 2.5s | 14 | 直线高速弹，非追踪 |
| 剑魔主体 | 召唤随从 | 1.5s | 10s | - | 仅随从全灭后触发 |
| 随从 | 齐射 | 0.25s | 1.2s | 10 | 保持中远距骚扰 |

**已拍板硬规则**：
- 随从再召唤最多 2 次（整场上限）。
- 若场上无随从，剑魔剑气冷却 2.5s -> 2.0s（防止空窗过长）。

**体验目标**：
- 玩家有明确优先级决策：先清随从还是先压本体。

---

## 5.3 丹魔（`segment_boss_3`）

**定位**：区域经营型 Boss。  
**主机制**：毒爆点杀 + 毒环控场。  
**阶段阈值**：40%。

| 单位 | 招式 | 前摇 | 冷却 | 伤害 | 细节 |
|------|------|------|------|------|------|
| 丹魔主体 | 毒爆 | 0.4s | 2.2s | 20 | 玩家脚下点名，半径 65 |
| 丹魔主体 | 毒环 | 0.5s | 3.5s | 12 | 40->80->40 扩散/回收 |
| 丹魔主体 | 邪灵强化 | - | - | - | <40% 后邪灵移速 +20%、伤害 +15% |
| 邪灵 | 追踪弹 | 0.25s | 1.5s | 8(强化后9) | 低速追踪，持续逼位 |

**体验目标**：
- 玩家被迫处理“脚下危险”与“空间封锁”的叠加，而非单纯拉扯。

---

## 5.4 秽源（`final_boss`）

**定位**：终局压制型 Boss。  
**主机制**：秽弹幕 + 邪气再生。  
**阶段阈值**：50%。

| 单位 | 招式 | 前摇 | 冷却 | 伤害 | 细节 |
|------|------|------|------|------|------|
| 秽源主体 | 秽冲 | 0.5s | 2.5s | 24 | 冲刺 0.35s，速度 420 |
| 秽源主体 | 秽弹幕 | 0.4s | 3.0s | 10 x 5 | 45 度扇形，弹速 220 |
| 秽源主体 | 邪气再生 | 1.0s | 7s/5s | - | 邪气死亡后再生 1 只 |
| 邪气 | 齐射 | 0.25s | 1.0s | 12 | 中远距持续施压 |

**已拍板硬规则**：
- 邪气再生整场最多 2 次。
- P2（<50%）后：移速 +15%，技能冷却 -20%，再生间隔 7s -> 5s。

**体验目标**：
- 最终战必须是“资源管理 + 站位管理 + 目标切换”的综合考试。

## 六、数值预算与难度护栏

> 对齐 `docs/BALANCE_DISCUSSION.md` 的生存体验目标。

### 6.1 生存预算（Boss 段）

- 玩家基础血量 100。
- 单次高危技能伤害建议在 18~24（已符合当前方案）。
- 不允许“单段不可避技能 + 两段连续命中 = 必死”。

### 6.2 战斗时长预算（单 Boss）

| Boss | 目标时长 |
|------|----------|
| 妖王 | 70~100 秒 |
| 剑魔 | 90~130 秒 |
| 丹魔 | 90~130 秒 |
| 秽源 | 120~180 秒 |

### 6.3 失败归因分布（目标）

- 60%：站位失误（可学习）
- 25%：资源管理失误（技能/CD/药）
- <=15%：随机因素（不可控）

## 七、实现架构建议（代码落地）

### 7.1 统一创建入口（已开始）

- `create_enemy(..., boss_id, enemy_index)` 统一处理 Boss 映射。
- `BOSS_REGISTRY` 管理 `boss_id -> BossClass`。

### 7.2 Boss 基类能力（建议抽象）

- 公共字段：`phase`, `cooldowns`, `state`, `state_timer`。
- 公共方法：`enter_phase()`, `can_cast()`, `cast_skill()`, `telegraph()`。
- 目标：后续 Boss 实现以“配置 + 小量覆写”为主。

### 7.3 数据配置建议

可逐步从 `data/levels.json` 拆出 `data/bosses.json`，结构示例：

```json
{
  "segment_boss_2": {
    "boss_id": "jianmo",
    "phase_thresholds": [0.5],
    "skills": {
      "sword_wave": {"windup": 0.3, "cooldown": 2.5, "damage": 14}
    }
  }
}
```

### 7.4 当前代码调参入口（已实现）

Boss 关键参数已集中到 `enemy.py` 的 `BOSS_TUNING`，用于快速调手感：

```python
BOSS_TUNING = {
    "segment_boss_1": {"skills": {"pounce": {"cooldown": 2.2}, ...}},
    "segment_boss_2": {"skills": {"wave_cd": 2.5, "summon_limit": 2}},
    "segment_boss_3": {"skills": {"toxic_ring": {"cooldown": 3.5}}},
    "final_boss": {"skills": {"regen": {"limit": 2, "cooldown": 7.0}}},
}
```

推荐调参顺序：

1. 先调 `cooldown`，控制压迫节奏。  
2. 再调 `damage`，控制容错。  
3. 最后调 `limit`/`threshold`，控制阶段结构。  

注意：优先改 `BOSS_TUNING`，避免直接改 Boss 类内部硬编码逻辑。

## 八、验收标准（Definition of Done）

每个 Boss 必须同时通过以下检查：

1. **机制验收**：主机制可稳定触发，阶段切换可复现。
2. **可读验收**：所有高伤技能有前摇且可被观察。
3. **公平验收**：无“必吃连段”与无限召唤。
4. **性能验收**：弹幕与召唤高峰时帧率稳定（无明显卡顿）。
5. **体验验收**：至少 3 局实测，失败原因可归类并可修正。

## 十一、Boss 难度试玩（仅 Boss）

为快速调 Boss 难度，已提供独立试玩入口：

```bash
python -m tools.boss_practice segment_boss_1
python -m tools.boss_practice segment_boss_2
python -m tools.boss_practice segment_boss_3
python -m tools.boss_practice final_boss
```

试玩默认构筑（非空手）：

- 灵根：火灵根
- 法宝：赤炎剑（sword）
- 丹药：沿用当前局外库存与上限规则

如需测试其他构筑，可先在村子切换灵根/法宝后再进入正常流程，或后续扩展 `boss_practice` 参数。

建议每次调参后只测 Boss：

1. 单 Boss 连续打 2~3 局，记录死亡原因。  
2. 重点看“是否可读、是否可反制、是否有喘息窗口”。  
3. 先调 `BOSS_TUNING` 的 `cooldown`，再调 `damage`，最后调 `limit`。
