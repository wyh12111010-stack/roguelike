# 玩家能力逻辑规范（Balanced）

> 目标：明确主角在战斗中的能力循环、窗口管理与反制关系。  
> 口径与 `docs/BALANCE_DISCUSSION.md`、`docs/PLAYER_ATTR_SOURCES.md` 一致。

---

## 一、玩家战斗定位

- **定位**：高机动中等容错的持续输出者。  
- **核心手段**：走位 + 法宝节奏 + 元素构筑。  
- **失败主要原因应是执行问题**，而不是“缺数值”。

---

## 二、能力循环（必须成立）

### 2.1 三循环模型

1. **输出循环**：普攻/法宝命中 -> 触发倍率/反应 -> 形成击杀。  
2. **生存循环**：读前摇 -> 走位/位移 -> 回避高危窗口。  
3. **资源循环**：灵力回复 -> 技能消耗 -> 关键时机释放。  

如果任一循环失效，会出现“伤害虚无”或“莫名暴毙”。

### 2.2 循环节奏目标

| 循环 | 目标感受 |
|------|----------|
| 输出 | 0.4~0.8s 有一次有效操作反馈 |
| 生存 | 每次高危前至少有一次可反应信号 |
| 资源 | 每 8~15s 有一次“是否交技能”的决策点 |

---

## 三、玩家能力模板（统一格式）

### 3.1 通用模板

| 维度 | 内容 |
|------|------|
| 定位 | 该能力在战斗中解决什么问题 |
| 前摇 | 玩家启动该能力需要的准备时间 |
| 执行 | 生效帧、伤害或效果区间 |
| 后摇 | 结束后的风险窗口 |
| 冷却/消耗 | 资源约束 |
| 克制关系 | 克制哪些敌方行为 |
| 风险点 | 使用错误会导致什么后果 |

### 3.2 当前能力映射

| 能力 | 定位 | 风险点 |
|------|------|--------|
| 普攻 | 稳定输出 | 贪刀导致吃高危技 |
| 法宝技能（规划中） | 爆发/解围 | 交错时机导致空蓝 |
| 走位 | 核心生存 | 路线被 AOE 封死 |
| 受击无敌（0.8s） | 防连段 | 误判无敌结束时间 |

---

## 四、玩家成长逻辑

### 4.1 局内成长（短局）

- 以“选择强化”为主，不应出现数值无限膨胀。
- 推荐来源：商店、事件、Boss奖励。
- 成长曲线：前期补手感，中期补功能，后期补上限。

### 4.2 局外成长（长线）

- 当前 `base_health_bonus`、`base_mana_bonus` 保持预留状态。
- 仅当通关率长期偏低时，作为柔性调节阀启用。

---

## 五、反模式（必须避免）

1. 纯堆伤害导致“秒怪或刮痧”两极化。  
2. 过多保命导致敌方机制失去意义。  
3. 技能耗蓝过高导致“只剩平A”。  
4. 反馈不足导致玩家无法判断操作是否正确。  

---

## 六、平衡验收（玩家视角）

- 普通关：玩家应能通过走位规避主要伤害。  
- Boss 关：玩家需至少两次资源决策（交技能/留技能）。  
- 失败复盘：可清晰归因到“贪刀、错位、错时机、错资源”。  

---

## 七、与灵根/法宝的关系

- 灵根负责“构筑方向”（倍率与反应倾向）。  
- 法宝负责“手感与节奏”（攻击方式、CD、技能形态）。  
- 两者结合形成流派，但不应出现唯一最优解。
